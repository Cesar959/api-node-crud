Sobre

Interface de Programação de Aplicações (português) cuja sigla API provém do Inglês Application Programming Interface, é um conjunto de rotinas e padrões estabelecidos por um software para a utilização das suas funcionalidades. Com o intuito de aprender como desenvolver uma api utilizando o php sem framework.

Introdução

API conta com 5 funcionalidades básicas: obter todos os registros, selecionar um unico registro, cadastrar registro, atualizar o registro e deletar o registro. Até o momento não foi implementado um sistema de autenticação para utilizar a API, possibilitando assim a utilização pública da mesma.

Tecnologias

  • PHP 7
  • URL amigável
  • SQL
  • Json

Metodos

POST - utilizado para cadastrar

GET - utilizado para pegar os dados

UPDATE - utilizado para atualizar

DELETE - utilizado para deletar

Rotas

É a mesma rota para todos o que diferencia é o método, para GET e DELETE é utilizado o id_usuarios do registro como parâmetro. Alem disso o unico serviço disponivel até o momento é "usuarios".

Cadastra os dados no banco de dados

POST

http://localhost/api_crud/usuarios/

Traz todos os registros

GET

http://localhost/api_crud/usuarios/

Traz um registo específico (o caso em questão, vai trazer os dados referente ao id_usuarios 2)

GET

http://localhost/api_crud/usuarios/2

Atualiza os dados no banco de dados. Aviso: na atualização do registro é obrigatorio o envio de todos os dados (id_usuarios, nome, idade, sexo, email, senha) mesmo aqueles que não sofrerão alteraçãoes.

PUT

http://localhost/api_crud/usuarios/

Excluir o registro baseado no id_usuarios (o caso em questão, vai excluir os dados referente ao id_usuarios 2)

DELETE

http://localhost/api_crud/usuarios/2

Mensagens de Status

200 : Requisição foi feita com sucesso.

404: Falha na requisição, possivelmente por falta de envio de dados ou os dados solicitados não existe

400: Recuso não localizado

500: Erro interno do servidor.
